# Configuration

## Config file locations

| File | Purpose |
|------|---------|
| `~/.swampcastle/config.json` | Global configuration |
| `~/.swampcastle/identity.txt` | L0 identity text (plain text) |
| `~/.swampcastle/wing_config.json` | Wing definitions and keywords |
| `~/.swampcastle/people_map.json` | Name variant → canonical name mappings |

All files are created by `swampcastle init`. The config directory permissions are set to `0o700` (owner only).

## config.json

```json
{
  "palace_path": "/home/user/.swampcastle/palace",
  "collection_name": "swampcastle_drawers",
  "topic_wings": ["emotions", "consciousness", "memory", "technical", "identity", "family", "creative"],
  "hall_keywords": {
    "emotions": ["scared", "afraid", "happy", "sad", "love", "feel"],
    "technical": ["code", "python", "bug", "error", "api", "database"],
    "memory": ["memory", "remember", "forget", "recall", "archive"]
  }
}
```

### Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `palace_path` | string | `~/.swampcastle/palace` | ChromaDB storage directory |
| `collection_name` | string | `swampcastle_drawers` | ChromaDB collection name |
| `topic_wings` | list | see above | Default topic wing names |
| `hall_keywords` | dict | see above | Keywords that map content to halls |
| `people_map` | dict | `{}` | Name variant mappings (alternative to `people_map.json`) |

## Environment variables

Environment variables take precedence over config file values.

| Variable | Overrides | Description |
|----------|-----------|-------------|
| `MEMPALACE_PALACE_PATH` | `palace_path` | Path to the palace directory |
| `MEMPAL_PALACE_PATH` | `palace_path` | Alias for `MEMPALACE_PALACE_PATH` |
| `MEMPALACE_BACKEND` | `backend` | Storage backend: `lance` or `chroma` |
| `MEMPAL_DIR` | (hooks only) | Directory for auto-ingest during hook saves |

## Precedence

Configuration values are resolved in this order (first match wins):

1. **CLI `--palace` flag** (sets `MEMPALACE_PALACE_PATH` env var)
2. **Environment variables** (`MEMPALACE_PALACE_PATH`, `MEMPAL_PALACE_PATH`)
3. **Config file** (`~/.swampcastle/config.json`)
4. **Defaults** (hardcoded in `config.py`)

## identity.txt

Plain text file describing who the AI is. Becomes Layer 0 — loaded every session via `wake-up`.

```
I am Atlas, a personal AI assistant for Alice.
Traits: warm, direct, remembers everything.
People: Alice (creator), Bob (Alice's partner).
Project: A journaling app that helps people process emotions.
```

There is no required format. Write whatever you want the AI to know about itself on startup.

## wing_config.json

Generated by `swampcastle init`. Maps projects and people to wings with keyword detection.

```json
{
  "default_wing": "wing_general",
  "wings": {
    "wing_kai": {"type": "person", "keywords": ["kai", "kai's"]},
    "wing_driftwood": {"type": "project", "keywords": ["driftwood", "analytics", "saas"]}
  }
}
```

## people_map.json

Maps name variants to canonical names. Used during mining to normalize references.

```json
{
  "kai": "Kai",
  "kai's": "Kai",
  "KAI": "Kai"
}
```

## MCP server configuration

The MCP server accepts a `--palace` flag to override the palace path:

```bash
python -m swampcastle.drawbridge --palace /custom/path/to/palace
```

The knowledge graph database is placed alongside the palace when `--palace` is used, at `<palace_path>/knowledge_graph.sqlite3`. Without `--palace`, it defaults to `~/.swampcastle/knowledge_graph.sqlite3`.

## Input validation

All wing names, room names, and entity names are validated by `sanitize_name()`:

- Must be a non-empty string
- Maximum 128 characters
- No path traversal characters (`..`, `/`, `\`)
- No null bytes
- Must match: `^[a-zA-Z0-9][a-zA-Z0-9_ .'-]{0,126}[a-zA-Z0-9]?$`

Content is validated by `sanitize_content()`:

- Must be a non-empty string
- Maximum 100,000 characters
- No null bytes

Invalid input raises `ValueError` (Python API) or returns `{"success": false, "error": "..."}` (MCP).

## Programmatic access

```python
from swampcastle.config import CastleConfig

config = CastleConfig()
config.palace_path        # resolved palace path
config.collection_name    # table/collection name
config.backend            # "lance" or "chroma"
config.embedder_config    # {"embedder": "...", "embedder_options": {...}}
config.node_id            # this machine's sync node ID
config.people_map         # name variant dict
config.topic_wings        # list of topic wing names
config.hall_keywords      # hall → keyword list dict

# Create config directory and default config.json
config.init()

# Save a people map
config.save_people_map({"kai": "Kai", "kai's": "Kai"})
```

Override the config directory (useful for testing):

```python
config = CastleConfig(config_dir="/tmp/test_swampcastle")
```
