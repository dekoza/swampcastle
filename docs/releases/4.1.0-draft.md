# SwampCastle 4.1.0

## Highlights

### Proposal-first knowledge graph extraction
SwampCastle 4.1.0 adds a full proposal-first knowledge graph extraction pipeline instead of writing extracted facts directly into the accepted KG.

This release adds:
- candidate-triple storage and review flow
- `swampcastle kg extract`
- `swampcastle kg review`
- `swampcastle kg accept`
- `swampcastle kg reject`
- conflict-aware review with `swampcastle kg review --conflicts-only`
- accept-with-invalidation via `--invalidate-conflicts`
- edit-before-accept overrides for subject / predicate / object
- optional gather-time proposal extraction with `swampcastle gather --extract-kg-proposals`
- a labeled fixture corpus and first extractor precision / recall regression gate

The accepted KG stays cleaner because extraction remains proposal-only until explicitly reviewed.

### Embedder contract hardening for multi-device sync
4.1.0 hardens the canonical embedding path for safer workstation + laptop sync:
- canonical ONNX path pinned to `CPUExecutionProvider`
- embedder fingerprints stored and checked
- Lance collections reject mixed embedder fingerprints, not just mismatched dimensions
- `swampcastle embedders --verify` reports fingerprint + deterministic probe hash for cross-machine comparison

Recommended canonical offline-safe config:

```json
{
  "embedder": "onnx"
}
```

### Better search and indexing ergonomics
This release improves retrieval and maintenance workflows:
- optional lexical reranking over dense search results
- lightweight hybrid retrieval via lexical candidate generation
- better chunking boundaries and overlap behavior
- live progress output for `swampcastle reforge` / `swampcastle reindex`
- adaptive reindex batching to avoid the worst slowdown from tiny progress batches

### Reliability and correctness fixes
4.1.0 also rolls up a large set of correctness and operational fixes:
- drawer ID collision fix and hash-input ambiguity fix
- embedder cache thread-safety fix
- sync server / client bearer-token auth support
- diary read performance fix
- SQLite KG concurrency fix
- graph summary cache + invalidation
- miner status truncation fix
- query sanitization hardening

## Upgrade notes

### Embeddings
If two machines are expected to sync precomputed vectors without re-embedding, verify they match before relying on that contract:

```bash
swampcastle embedders --verify --json
```

Both machines should report the same:
- `fingerprint_hash`
- `probe_hash`

If you intentionally change embedder, re-embed explicitly:

```bash
swampcastle reindex
```

### Knowledge graph extraction
KG auto-extraction remains proposal-first. Gather-time extraction is opt-in:

```bash
swampcastle gather <dir> --extract-kg-proposals
```

This does **not** auto-write accepted facts into the KG.
